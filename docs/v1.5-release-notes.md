# V1.5.0 Release Notes

V1.5.0 upgrades retrieval quality while preserving the existing `VECTOR`, `HYBRID`, and `RERANK` execution modes. `CLASSIC` remains the unconditional default; every non-classic knowledge-base default requires passing evidence for the current index revision.

## Highlights

- Parent-Child indexing retrieves high-recall child chunks and returns the complete parent context.
- QA-assisted indexing generates bounded candidate question/answer entries and expands QA hits back to their source chunks.
- Retrieval profiles add `CLASSIC`, `PARENT_CHILD`, `QA_ASSISTED`, and `COMBINED` knowledge-base-level selection.
- Profile V2 uses one filterable Milvus superset so all profiles compare equivalent document revisions.
- Combined retrieval applies weighted RRF across child and QA routes before optional reranking.
- Revision-bound quality gates block non-classic defaults when hit rate or citation quality regresses against `CLASSIC`.
- Knowledge-base settings and RAG evaluation pages expose Profile V2 readiness, gate decisions, rate deltas, and blocking reasons.

## Compatibility

- Existing clients that omit a retrieval profile continue to use `CLASSIC`.
- Existing `VECTOR`, `HYBRID`, and `RERANK` behavior is not replaced.
- QA generation failure degrades to the source-document path without failing ingestion.
- Profile changes do not trigger another rebuild after the shared Profile V2 index is ready.
- V1.4 upload governance, recovery, sparse migration, RBAC, audit, and reliable queue contracts remain included.

## Upgrade Notes

- Flyway advances through V18, V19, and V20.
- Configure the same dedicated `MILVUS_PROFILE_COLLECTION` in API and Worker.
- Existing knowledge bases need one rolling reindex to reach `index_schema_version=2` and establish the current `index_revision`.
- Do not activate a non-classic default until Profile V2 readiness is complete and the candidate has a current `PASSED` quality decision.
- Read [the V1.5.0 release runbook](v1.5-release-runbook.md) before deployment, especially the Legacy cleanup and rollback constraints.

## Defaults And Limitations

- V1.5.0 does not globally enable a new retrieval profile.
- A quality decision becomes stale when the knowledge-base index revision changes.
- The minimum gate is three citation-evaluable cases; production rollout should use a larger representative corpus.
- Combined weights remain `child=1.0`, `qa=0.8`, `RRF_K=60` until retained benchmark evidence supports tuning.
- The 22 security exceptions expire on 2026-08-15 and must be re-scanned or removed before that date.
