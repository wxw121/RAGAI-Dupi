# V1.4.2 Governance Ops Design

> **中文说明：** 本设计为 V1.4.1 上传与摄取治理层增加只读的统一运维摘要和冒烟演练，帮助运维人员快速判断队列与账本是否健康。

Date: 2026-07-18
Status: In development

## Scope

V1.4.2 is a stabilization release for V1.4.1 upload and ingest governance. It adds one read-only operational summary endpoint and a smoke rehearsal so an operator can quickly inspect whether the governance queues and ledgers are healthy after deploy.

This version intentionally avoids new indexing modes, SSO/OIDC, new notification transports, and broad Web UI changes.

## Operator Problem

V1.4.1 introduced durable upload quota reservations, leased ingest jobs, transactional ingest outbox events, failure notifications, and vector cleanup tasks. Those systems expose detailed APIs and alerts, but an operator still needs one compact view to answer:

- Are upload quota reservations stuck in stale pending attempts?
- Are ingest jobs processing with expired leases?
- Are outbox events or failure notifications stuck?
- Are vector cleanup tasks accumulating?
- What existing audit, ingest, and vector alerts are active?

## API Contract

Endpoint: GET /api/v1/ops/governance-summary

The route lives under the existing /api/v1/ops authorization policy and is restricted to OPS_ADMIN operators.

Response sections: generatedAt, uploadQuota, ingestJobs, ingestOutbox, failureNotifications, vectorCleanup, alerts.

The response is read-only. It does not mutate reservations, jobs, outbox rows, notification rows, cleanup tasks, or audit logs.

## Aggregation Model

GovernanceOpsService aggregates upload quota, ingest jobs, ingest outbox, failure notifications, vector cleanup, and existing alert services.

V1.4.2 appends alert codes for stale upload attempts, expired processing leases, failed outbox rows, and exhausted failure notifications when counts are above zero.

## Smoke Rehearsal

Smoke command: powershell -NoProfile -ExecutionPolicy Bypass -File scripts/smoke-governance-summary.ps1 -BaseUrl http://localhost:8080 -ApiKey $env:DUPI_API_KEY -OutFile evidence/governance-summary-smoke.json

The script validates the JSON shape, writes redacted evidence, and never prints the API key. Pester coverage verifies parameters, endpoint path, required response fields, sample validation, and redacted evidence behavior.

## Local Environment Note

The local Web toolchain currently runs on Node 16. V1.4.2 verification must use the existing npm scripts so services/web/scripts/node16-webcrypto.cjs loads the WebCrypto shim. Do not run raw vite or vitest directly in this environment.

## Non Goals

- No Parent Child or QA indexing.
- No SSO/OIDC or external identity provider sync.
- No new email or IM notification channel.
- No governance dashboard UI beyond the existing ops API surface.
- No release, tag, or merge claim in this design document.
