# V1.1 Observability Evaluation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use $superpower-subagents (recommended) or $superpower-executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking via update_plan.

**Goal:** Ship the V1.1 small-version upgrade that adds a real-browser E2E gate, ingest/index diagnosis, a lightweight RAG evaluation loop, and P1 upload/ops visibility.

**Architecture:** Keep this as a thin closure layer on existing flows. Backend diagnosis and guardrails are computed from existing entities/configuration, frontend evaluation reuses `/retrieve`, and browser E2E runs against the same login/cookie/CSRF path that users exercise.

**Tech Stack:** Spring Boot 3, Java 17, Maven, React 18, Vite, TypeScript, Vitest, Playwright, PowerShell.
