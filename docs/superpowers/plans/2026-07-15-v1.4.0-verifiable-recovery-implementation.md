# V1.4.0 Verifiable Recovery Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use $superpower-subagents (recommended) or $superpower-executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking via update_plan.

**Goal:** Add tenant-isolated, checksum-verified knowledge-base archives and idempotent restore into a new hidden knowledge base.

**Architecture:** Recovery metadata and item progress live in PostgreSQL, archive payloads live under a sealed prefix in a dedicated private MinIO bucket, and dense/sparse vectors are streamed through a focused Milvus snapshot adapter. A background executor advances explicit archive and restore state machines; stable item keys and deterministic target identifiers make retry safe across process or dependency failures.

**Tech Stack:** Java 17, Spring Boot/JPA/Flyway, MinIO Java SDK, Milvus Java SDK, PostgreSQL, React/Vite/Vitest, Playwright, PowerShell.
