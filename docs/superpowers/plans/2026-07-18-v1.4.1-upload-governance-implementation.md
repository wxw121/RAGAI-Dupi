# V1.4.1 Upload Governance Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use $superpower-subagents (recommended) or $superpower-executing-plans to implement this plan task-by-task. Steps use checkbox (`- [x]`) syntax for tracking via update_plan.

**Goal:** Ship persisted upload quota, reliable ingest execution/cancellation, stale callback protection, failure notification outbox, and governed per-file uploads.

**Architecture:** PostgreSQL becomes the authority for quota reservations and ingest execution state; Redis remains a recoverable transport. Worker callbacks carry execution IDs and sequences. Web uploads each file independently with visible quota and per-file control.

**Tech Stack:** Spring Boot 3.2/JPA/Flyway/PostgreSQL/Redis, Python FastAPI/redis/httpx/pytest, React/Vite/Vitest, Docker Compose, local release scan scripts.
