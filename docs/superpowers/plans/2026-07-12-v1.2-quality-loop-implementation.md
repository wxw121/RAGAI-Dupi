# V1.2 Quality Loop Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use $superpower-subagents (recommended) or $superpower-executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking via update_plan.

**Goal:** Upgrade dupi-RAG to V1.2 with stronger browser regression coverage, explainable upload/index details, structured chat errors, managed RAG evaluation, and P1 operational/product extensions.

**Architecture:** Keep the current Spring Boot API, React web console, Python worker, and Playwright gate. Add small vertical slices: API DTO/service/controller support first, then UI surfaces, then E2E coverage and documentation. Persist RAG evaluation cases/results in PostgreSQL because they are user-owned quality assets.

**Tech Stack:** Spring Boot 3.2, JPA/Flyway/PostgreSQL, React/Vite/Vitest, Playwright, Python worker hybrid retrieval.
