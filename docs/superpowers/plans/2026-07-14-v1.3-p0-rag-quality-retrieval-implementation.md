# V1.3 P0 RAG Quality Gate and Retrieval Implementation Plan

> **中文说明：** 本实施计划增加可阻断发布的 RAG 质量基线，并把混合检索从请求时内存 BM25 迁移到带版本的 Milvus 稀疏检索配置。

> For agentic workers: use superpower-executing-plans task-by-task. Steps use checkbox syntax for update_plan.

**Goal:** Add a blockable RAG quality baseline and migrate hybrid retrieval from request-time in-memory BM25 to a versioned Milvus sparse profile.

**Architecture:** Evaluation runs persist immutable metrics, case fingerprints, comparison state, and profile snapshots. A per-KB policy and explicit baseline produce the gate result. A new versioned sparse collection is backfilled and dual-written before cutover.

**Tech Stack:** Spring Boot/JPA/Flyway, Milvus, Python/FastAPI, React/Vite/Vitest, Playwright, PowerShell.

---
