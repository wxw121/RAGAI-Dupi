# V1.5 RAG Quality Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use $superpower-executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking via update_plan.

**Goal:** Add retrieval profiles for classic, parent-child, QA-assisted, and combined retrieval, with knowledge-base-level selection and RAG evaluation comparison gates.

**Architecture:** Keep the existing VECTOR/HYBRID/RERANK retrieval path intact and layer profiles on top through request/profile parameters. Store parent-child and QA provenance in chunk metadata first, using explicit profile diagnostics so behavior remains reversible and explainable.

**Tech Stack:** Spring Boot API, PostgreSQL/Flyway, Milvus, Python worker retrieval/indexing, React/Vite web UI, JUnit/Mockito, pytest, Vitest.

## Execution Status (2026-07-19)

- [x] P0 profile persistence, request override, parent-child indexing/expansion, multi-profile eval, and minimal Web controls implemented.
- [x] Parent-child ingest stores parents in PostgreSQL, indexes children in Milvus, and excludes parents from BM25 entry retrieval.
- [x] P1 automatic QA generation, persisted QA source provenance, non-fatal ingest fallback, and QA-assisted/combined source expansion implemented.
- [x] P2 profile-isolated/filterable v2 index superset, Combined weighted RRF tuning, automated default-enable quality gate, and readiness/gate comparison visualization implemented.
