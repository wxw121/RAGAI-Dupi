# V1.5 P2 Profile-Isolated Quality Gate Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use $superpower-subagents (recommended) or $superpower-executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking via update_plan.

**Goal:** Build a filterable all-profile index, deterministic Combined fusion, revision-bound RAG quality metrics, and an enforced default-profile gate.

**Architecture:** Every ingest creates one reusable superset of original, parent, child, and two QA entry groups. A new Milvus v2 collection filters candidates before ranking, while the legacy collection remains readable until a knowledge base finishes migration. Evaluation stores current-revision metrics and gate decisions; non-classic default changes require a current `PASSED` decision.

**Tech Stack:** Spring Boot/JPA/Flyway/Jackson, Milvus Java SDK and pymilvus, Python/httpx/rank-bm25, React/Vite, JUnit/Mockito, pytest, Vitest.

```python
@dataclass
class ProfileIndexPlan:
    persisted_chunks: list[TextChunk]
    v2_index_chunks: list[TextChunk]
    legacy_chunks: list[TextChunk]

def build_profile_index_plan(nodes, chunk_size, chunk_overlap, strategy, embed_fn,
                             kb_id, doc_id) -> ProfileIndexPlan:
    original_chunks = chunk_nodes(
        nodes, chunk_size=chunk_size, chunk_overlap=chunk_overlap,
        strategy=strategy, embed_fn=embed_fn,
    )
    parent_chunks, child_chunks = build_parent_child_chunks(
        nodes, child_chunk_size=chunk_size, child_overlap=chunk_overlap,
        strategy=strategy, embed_fn=embed_fn,
    )
    mark_scope(original_chunks, "original", ["classic", "qa-assisted"])
    mark_scope(parent_chunks, "parent", ["parent-child", "combined"])
    mark_scope(child_chunks, "child", ["parent-child", "combined"])
    original_qa = generate_qa_chunks(
        kb_id, doc_id, original_chunks, start_index=0,
        profile_scope=["qa-assisted"],
    )
    parent_qa = generate_qa_chunks(
        kb_id, doc_id, parent_chunks, start_index=0,
        profile_scope=["combined"],
    )
    persisted = [*original_chunks, *parent_chunks, *child_chunks, *original_qa, *parent_qa]
    for index, chunk in enumerate(persisted):
        chunk.chunk_index = index
    return ProfileIndexPlan(
        persisted_chunks=persisted,
        v2_index_chunks=[*original_chunks, *child_chunks, *original_qa, *parent_qa],
        legacy_chunks=original_chunks,
    )
```

Build original and parent/child chunks once. Set `entry_kind` and `profile_scope` metadata before QA generation. Extend `generate_qa_chunks` with an explicit `profile_scope` argument so inherited source scope cannot leak into QA entries. Reassign every persisted chunk index contiguously after assembling all groups.

Extend Redis normalization for `legacyWriteRequired` and `indexSchemaVersion`; default `legacyWriteRequired` to `true` until the API starts emitting the field in Task 4.

```python
def weighted_rrf(routes: list[tuple[float, list[dict]]], k: int = 60) -> list[dict]:
    if k < 1:
        raise ValueError("RRF K must be at least 1")
    scores: dict[str, float] = {}
    items: dict[str, dict] = {}
    for weight, hits in routes:
        if weight <= 0:
            raise ValueError("route weight must be positive")
        for rank, hit in enumerate(hits):
            chunk_id = hit["chunk_id"]
            scores[chunk_id] = scores.get(chunk_id, 0.0) + weight / (k + rank + 1)
            items[chunk_id] = hit
    return [
        {**items[chunk_id], "score": score}
        for chunk_id, score in sorted(scores.items(), key=lambda item: item[1], reverse=True)
    ]
```

Non-combined profiles retain existing vector/BM25 RRF behavior over their isolated corpus. Combined builds four routes and applies child/QA weights before optional reranking.
