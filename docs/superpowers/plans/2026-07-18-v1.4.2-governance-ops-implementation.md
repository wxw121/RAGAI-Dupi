# V1.4.2 Governance Ops Implementation Plan

Date: 2026-07-18
Status: In development

## Goal

Add one read-only governance operations summary for V1.4.1 state, plus smoke automation and documentation. Keep the release narrow and stabilizing.

## Work Plan

1. Baseline and branch isolation
   - Use worktree .worktrees/v1.4.2-governance-ops.
   - Keep .npm-cache ignored for local npm installs.
   - Preserve V1.4.1 behavior and avoid broad Web changes.

2. API summary contract
   - Add GovernanceSummaryResponse with generatedAt, uploadQuota, ingestJobs, ingestOutbox, failureNotifications, vectorCleanup, and alerts.
   - Add GovernanceOpsService to aggregate existing repositories and alert services.
   - Add read-only repository counters only.
   - Expose GET /api/v1/ops/governance-summary from OpsController.

3. Tests
   - Add GovernanceOpsServiceTest first to lock aggregation and alert ordering.
   - Update controller layer coverage for the new ops route.
   - Focused API gate: mvn.cmd -q -f services/api/pom.xml -Dtest=GovernanceOpsServiceTest,ControllerLayerTest test.

4. Smoke automation
   - Add scripts/smoke-governance-summary.ps1.
   - Validate required top-level fields.
   - Write redacted evidence to an operator-selected path.
   - Support sample validation for local Pester tests.
   - Pester gate: powershell -NoProfile -ExecutionPolicy Bypass -Command Import-Module Pester; Invoke-Pester -Path scripts/tests/smoke-governance-summary.Tests.ps1 -CI.

5. Documentation
   - Add this implementation plan.
   - Add the design spec.
   - Add docs/v1.4.2-governance-ops-runbook.md.
   - Sync README, architecture, progress, and todo.
