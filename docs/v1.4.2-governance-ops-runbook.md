# V1.4.2 Governance Ops Runbook

> **中文说明：** 本手册说明 V1.4.2 治理运维摘要接口的授权、响应结构、冒烟检查及健康状态判定方法。

Status: In development

V1.4.2 adds a compact read-only ops summary for the V1.4.1 upload and ingest governance layer.

## Endpoint

GET /api/v1/ops/governance-summary

Authorization follows the existing /api/v1/ops policy and requires an OPS_ADMIN operator session or API access path.

## Response Shape

The response contains these top-level sections: generatedAt, uploadQuota, ingestJobs, ingestOutbox, failureNotifications, vectorCleanup, alerts.

## Smoke Check

Run against a local or deployed API: powershell -NoProfile -ExecutionPolicy Bypass -File scripts/smoke-governance-summary.ps1 -BaseUrl http://localhost:8080 -ApiKey $env:DUPI_API_KEY -OutFile evidence/governance-summary-smoke.json

The script validates required top-level fields and writes redacted evidence. It records whether an API key was configured but does not print or write the key value.

## Pester Check

Command: powershell -NoProfile -ExecutionPolicy Bypass -Command Import-Module Pester; Invoke-Pester -Path scripts/tests/smoke-governance-summary.Tests.ps1 -CI

Current focused result: 4 passed, 0 failed.

## Interpreting Alerts

- UPLOAD_STALE_PENDING_ATTEMPTS: pending upload attempts have expired leases and need reconciliation attention.
- INGEST_EXPIRED_PROCESSING_LEASES: processing jobs have non-null expired leases.
- INGEST_OUTBOX_FAILURES_OPEN: ingest outbox rows are in FAILED status.
- INGEST_FAILURE_NOTIFICATIONS_EXHAUSTED: failure notification delivery reached max attempts.

The summary also includes existing audit, ingest, and vector cleanup alert responses in the same AuditAlertResponse shape used by /api/v1/ops/audit-alerts.
