# V1.5.0版本说明

<!-- language-switch -->
[English](../en/v1.5-release-notes.md)





> **中文说明：** V1.5.0 在保留现有检索执行模式的基础上提升 RAG 检索质量，并要求非经典检索配置通过当前索引修订版的质量门禁后才能成为默认配置。

V1.5.0升级了检索质量，同时保留了现有的“VECTOR”、“HYBRID”和“RERANK”执行模式。‘ CLASSIC ’仍然是无条件的默认值；每个非经典知识库默认值都需要为当前索引修订传递证据。

# #亮点

-亲子索引检索高召回子块并返回完整的父上下文。
- QA辅助索引生成有限的候选问题/答案条目，并将QA点击扩展回其源块。
-检索配置文件增加‘ CLASSIC ’， ‘ PARENT_CHILD ’， ‘ QA_ASSISTED ’和‘ COMBINED ’知识库级别选择。
-配置文件V2使用一个可过滤的Milvus超集，所以所有的配置文件比较等效的文档修订。
-组合检索在可选的重新排序之前对子路由和QA路由应用加权RRF。
-当点击率或引用质量回归到“经典”时，修订绑定质量门会阻止非经典默认值。
-知识库设置和RAG评估页面公开了配置文件V2准备情况、gate决策、速率增量和阻塞原因。

# #兼容性

-忽略检索配置文件的现有客户端继续使用“CLASSIC”。
-现有的‘ VECTOR ’， ‘ HYBRID ’和‘ RERANK ’行为不会被替换
- QA生成失败将降级到源文档路径，而不会摄取失败。
—共享配置文件V2索引准备好后，配置文件更改不会触发另一次重建。
—V1.4仍然包括上传治理、恢复、稀疏迁移、RBAC、审计和可靠队列契约。

##升级说明

- Flyway通过V18， V19和V20。
-在API和Worker中配置相同的专用‘ MILVUS_PROFILE_COLLECTION ’。
-现有的知识库需要一个滚动索引达到‘ index_schema_version=2 ’，并建立当前的‘ index_revision ’。
-在配置文件V2准备完成并且候选人当前的质量决定为“通过”之前，不要激活非经典默认值。
- 阅读 [V1.5.0 版本运行手册](v1.5-release-runbook.md)，特别是 Legacy 清理和回滚约束。

默认值和限制

- V1.5.0不全局启用新的检索配置文件。
-当知识库索引修订发生变化时，质量决策变得陈旧。
-最低门槛是三个可被引用的案例；生产部署应该使用更大的代表性语料库。
-组合权重保持‘ child=1.0 ’， ‘ qa=0.8 ’， ‘ RRF_K=60 ’，直到保留基准证据支持调优。
—22个安全例外将于2026-08-15过期，必须在该日期之前重新扫描或删除。
