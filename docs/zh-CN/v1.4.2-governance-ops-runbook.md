# #端点

得到/ api / v1 /行动/ governance-summary

授权遵循现有的/api/v1/ops策略，需要OPS_ADMIN操作员会话或api访问路径。

##响应形状

响应包含这些顶级部分：generatedAt， uploadQuota, ingestJobs, ingestOutbox, failureNotifications, vectorCleanup, alerts。

##检查烟雾

针对本地或部署的API运行：powershell -NoProfile -ExecutionPolicy Bypass -File scripts/smoke-govern -summary。ps1 -BaseUrl http://localhost:8080 -ApiKey $env:DUPI_API_KEY -OutFile evidence/ government -summary-smoke.json

该脚本验证所需的顶级字段并写入编校的证据。它记录是否配置了API密钥，但不打印或写入密钥值。

##骚扰检查

命令：powershell -NoProfile -ExecutionPolicy Bypass -Command Import-Module Pester；调用-纠缠-路径脚本/测试/烟雾治理-总结。ps1 ci

当前关注的结果：4个通过，0个失败。

##解释警报

—“UPLOAD_STALE_PENDING_ATTEMPTS”：待处理的上传尝试已经过期，需要注意对账。
—ingest_expired_processing_租约：处理任务的租约过期不为空。
—INGEST_OUTBOX_FAILURES_OPEN：表示取出行处于FAILED状态。
—INGEST_FAILURE_NOTIFICATIONS_EXHAUSTED：失败通知已达到最大尝试次数。

摘要还包括/api/v1/ops/audit-alerts使用的相同AuditAlertResponse形状的现有审计、摄取和矢量清理警报响应。

本地Web验证说明

这个工作站有Node 16。对于Web检查，使用项目npm脚本：services/ Web /scripts/node16-webcrypto。cjs加载WebCrypto shim。不要在Node 16上直接调用原始vite或vitest。
