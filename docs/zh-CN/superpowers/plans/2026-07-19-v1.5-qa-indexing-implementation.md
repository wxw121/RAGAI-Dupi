# V1.5 QA索引实施计划

<!-- language-switch -->
[English](../../../en/superpowers/plans/2026-07-19-v1.5-qa-indexing-implementation.md)





> **中文说明：** 本实施计划通过现有 API 大模型生成有界问答候选，持久化带来源信息的 QA 分块，并让 QA 辅助与组合检索返回源上下文。

> **对于代理工作者：**需要的子技能：使用$superpower- execution -plans逐个任务地执行此计划。步骤使用复选框（' -[]'）语法通过update_plan进行跟踪。

**目标：**通过现有的API LLM添加有界的QA候选生成，保留带有来源的QA块，并使' QA_ASSISTED ' / ' COMBINED '检索源上下文而不改变经典默认值。

架构：Worker仍然负责摄取解析、分块、嵌入和Milvus写入。它为批处理QA候选者调用经过身份验证的API内部端点；API重用LlmClient。并返回经过验证的候选人。QA块与‘ source_chunk_id ’元数据持久化，并在正常可搜索条目旁边索引；失败会退回到原始块。

**技术栈：** Spring Boot/Jackson/WebClient， PostgreSQL块元数据，Python/httpx, Milvus, JUnit/Mockito, pytest。
