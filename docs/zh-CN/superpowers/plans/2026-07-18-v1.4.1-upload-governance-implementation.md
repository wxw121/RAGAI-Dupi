# V1.4.1上传治理实施计划

<!-- language-switch -->
[English](../../../en/superpowers/plans/2026-07-18-v1.4.1-upload-governance-implementation.md)





> **中文说明：** 本实施计划交付持久化上传配额、可靠摄取与取消、过期回调防护、失败通知发件箱以及受治理的逐文件上传。

b> **对于代理工作者：**需要的子技能：使用$superpower-subagents（推荐）或$superpower- execution -plans逐个任务地执行此计划。步骤使用复选框（' - [x] '）语法通过update_plan进行跟踪。

**目标：** Ship持续上传配额，可靠的摄取执行/取消，过时的回调保护，失败通知发件箱，以及受管理的每个文件上传。

**架构：** PostgreSQL成为配额保留和摄取执行状态的权威；Redis仍然是一个可恢复的传输。工作回调带有执行id和序列。Web独立上传每个文件，具有可见的配额和每个文件控制。

**技术栈：** Spring Boot 3.2/JPA/Flyway/PostgreSQL/Redis, Python FastAPI/ Redis /httpx/pytest, React/Vite/Vitest, Docker Compose，本地发布扫描脚本。
