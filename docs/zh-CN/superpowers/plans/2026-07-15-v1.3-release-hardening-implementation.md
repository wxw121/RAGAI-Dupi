**目标：**在发布V1.3之前，为检索质量、重新排序延迟、Milvus升级恢复、供应链风险和稀疏迁移操作提供可重复的发布证据。

架构：在现有所有权边界内保持发布强化：Worker管理模型生命周期，PowerShell脚本编排基准测试/排练/扫描，知识库Web页面操作api拥有的迁移状态机。生成的证据写在“工件/”下；提交稳定的fixture、脚本、测试和运行本。

**技术栈：** Python/FastAPI/pytest，句子变压器/PyTorch， Spring Boot/JPA， React/Vite/Vitest/剧作家，Docker Compose, PowerShell, Milvus 2.5.4, pip-audit, Syft, Trivy。

def test_failed_warmup_is_reported_without_crashing_worker(monkeypatch):
    monkeypatch.setattr("app.reranker.CrossEncoder", Mock(side_effect=RuntimeError("missing weights")))

# #自我回顾

-该计划涵盖每一个已批准的设计部分，并在提交时保留生成的环境证据。
-状态和接受决定仍然属于服务器所有；Web面板不能强制进行无效的转换。

文档同步和“dev/”分支命名是明确的交付要求。
