# V1.2质量循环实施计划

<!-- language-switch -->
[English](../../../en/superpowers/plans/2026-07-12-v1.2-quality-loop-implementation.md)





> **中文说明：** 本实施计划通过浏览器回归、可解释上传/索引详情、结构化聊天错误、受管 RAG 评估与运维扩展，构建 V1.2 质量闭环。

b> **对于代理工作者：**需要的子技能：使用$superpower-subagents（推荐）或$superpower- execution -plans逐个任务地执行此计划。步骤使用复选框（' -[]'）语法通过update_plan进行跟踪。

**目标：**升级dupi-RAG到V1.2，具有更强的浏览器回归覆盖，可解释的上传/索引细节，结构化聊天错误，可管理的RAG评估和P1操作/产品扩展。

**架构：**保持当前的Spring Boot API、React web控制台、Python worker和剧作家门。添加小的垂直切片：首先是API DTO/服务/控制器支持，然后是UI表面，然后是端到端覆盖和文档。在PostgreSQL中持久化RAG评估用例/结果，因为它们是用户拥有的高质量资产。

**技术栈：** Spring Boot 3.2, JPA/Flyway/PostgreSQL, React/Vite/Vitest，剧作家，Python worker混合检索。

# #自我回顾

规格覆盖：所有请求的V1.2 P0和P1项目都有一个任务。
-占位符扫描：没有TBD或实现后的占位符保留。
-类型一致性：DTO名称和端点名称在后端、前端、测试和文档中都是稳定的。
-释放强化：Flyway V8轨道‘ RUNNING/COMPLETED/FAILED ’评估播种/创建是锁定保护的，上限为100例；文档预览和导出是有限制的；import validate ' schemaVersion=1 '；Webhook交付具有独立的权限、审计动作和可配置的超时。
-验证基线：API 240测试，JaCoCo线覆盖率为95.1913%；Web 57测试和生产构建；Docker V8迁移；《Chromium 1》以2.7s的核心流通过了测试（3.6s）。
