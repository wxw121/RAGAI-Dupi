# V1.5 RAG质量升级实施计划

<!-- language-switch -->
[English](../../../en/superpowers/plans/2026-07-18-v1.5-rag-quality-upgrade-implementation.md)





> **中文说明：** 本实施计划增加经典、父子、QA 辅助与组合检索配置，支持知识库级选择，并提供 RAG 评估比较门禁。

> **对于代理工作者：**需要的子技能：使用$superpower- execution -plans逐个任务地执行此计划。步骤使用复选框（' -[]'）语法通过update_plan进行跟踪。

**目标：**增加经典检索、亲子检索、qa辅助检索和组合检索的检索配置文件，具有知识库级选择和RAG评价比较门。

**架构：**保持现有的VECTOR/HYBRID/RERANK检索路径完整，并通过请求/配置文件参数将层配置文件放在顶部。首先在块元数据中存储父子和QA来源，使用显式配置文件诊断，使行为保持可逆和可解释。

**技术栈：** Spring Boot API， PostgreSQL/Flyway, Milvus， Python worker检索/索引，React/Vite web UI, JUnit/Mockito, pytest, Vitest。
